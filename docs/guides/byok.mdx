# BYOK (Bring Your Own Key)

BYOK allows you to use your own provider API keys while still benefiting from OpenRouter's unified interface.

## Overview

With BYOK, you can:
- Use your existing provider accounts
- Access models not available on OpenRouter's default providers
- Maintain direct billing relationships with providers

## List BYOK Credentials

```bash
curl https://openrouter.ai/api/v1/byok \
  -H "Authorization: Bearer $OPENROUTER_API_KEY"
```

## Create a BYOK Credential

```bash
curl -X POST https://openrouter.ai/api/v1/byok \
  -H "Authorization: Bearer $OPENROUTER_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "provider": "openai",
    "api_key": "sk-your-openai-key"
  }'
```

## Supported Providers

| Provider | Slug | Notes |
|----------|------|-------|
| OpenAI | `openai` | GPT-4, GPT-3.5 |
| Anthropic | `anthropic` | Claude models |
| Google | `google` | Gemini models |
| Cohere | `cohere` | Command models |
| Mistral | `mistral` | Mistral models |

## Get a BYOK Credential

```bash
curl https://openrouter.ai/api/v1/byok/{id} \
  -H "Authorization: Bearer $OPENROUTER_API_KEY"
```

## Update a BYOK Credential

```bash
curl -X PATCH https://openrouter.ai/api/v1/byok/{id} \
  -H "Authorization: Bearer $OPENROUTER_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "api_key": "sk-new-key"
  }'
```

## Delete a BYOK Credential

```bash
curl -X DELETE https://openrouter.ai/api/v1/byok/{id} \
  -H "Authorization: Bearer $OPENROUTER_API_KEY"
```

## Using BYOK in Requests

Once configured, BYOK credentials are automatically used for requests to models from that provider.

## Security

- API keys are encrypted at rest
- Keys are never returned in API responses
- Use management API keys for BYOK operations
