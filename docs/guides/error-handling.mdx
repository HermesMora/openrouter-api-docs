# Error Handling

OpenRouter uses conventional HTTP response codes and returns structured error objects.

## Error Format

All errors follow this structure:

```json
{
  "error": {
    "message": "Human-readable error description",
    "type": "error_category",
    "param": "parameter_name",
    "code": "error_code"
  }
}
```

## HTTP Status Codes

| Code | Meaning | Description |
|------|---------|-------------|
| 200 | OK | Request succeeded |
| 400 | Bad Request | Invalid request format or parameters |
| 401 | Unauthorized | Invalid or missing API key |
| 404 | Not Found | Model or resource not found |
| 429 | Rate Limited | Too many requests |
| 500 | Server Error | Internal server error |

## Common Error Codes

| Code | Description | Resolution |
|------|-------------|------------|
| `invalid_request_error` | Malformed request | Check request format |
| `invalid_api_key` | Authentication failed | Verify API key |
| `rate_limit_exceeded` | Too many requests | Wait and retry |
| `model_not_found` | Model doesn't exist | Check model ID |
| `insufficient_credits` | Out of credits | Add credits |

## Retrying Requests

For 429 or 5xx errors, implement exponential backoff:

```python
import time
import random

def retry_with_backoff(func, max_retries=3):
    for attempt in range(max_retries):
        try:
            return func()
        except RateLimitError:
            if attempt == max_retries - 1:
                raise
            delay = (2 ** attempt) + random.uniform(0, 1)
            time.sleep(delay)
```
