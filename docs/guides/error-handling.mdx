# Error Handling

OpenRouter returns standard HTTP status codes and structured error responses.

## Error Response Format

All errors follow this structure:

```json
{
  "error": {
    "message": "Human-readable error description",
    "code": 400
  }
}
```

## HTTP Status Codes

| Code | Meaning | Description |
|------|---------|-------------|
| 200 | OK | Success |
| 201 | Created | Resource created successfully |
| 202 | Accepted | Request accepted for processing |
| 400 | Bad Request | Invalid request parameters |
| 401 | Unauthorized | Missing or invalid API key |
| 402 | Payment Required | Insufficient credits |
| 403 | Forbidden | Insufficient permissions |
| 404 | Not Found | Resource does not exist |
| 408 | Request Timeout | Request timed out |
| 409 | Conflict | Resource conflict |
| 410 | Gone | Resource permanently removed |
| 413 | Payload Too Large | Request body too large |
| 422 | Unprocessable Entity | Validation error |
| 429 | Too Many Requests | Rate limit exceeded |
| 500 | Internal Server Error | Server error |
| 502 | Bad Gateway | Upstream provider error |
| 503 | Service Unavailable | Service temporarily unavailable |
| 524 | Edge Network Timeout | Cloudflare timeout |
| 529 | Provider Overloaded | AI provider is overloaded |

## Common Errors

### Rate Limit (429)

```json
{
  "error": {
    "message": "Rate limit exceeded. Please try again later.",
    "code": 429
  }
}
```

**Headers:**
- `Retry-After`: Seconds until the rate limit resets

### Insufficient Credits (402)

```json
{
  "error": {
    "message": "Insufficient credits. Please add more credits to your account.",
    "code": 402
  }
}
```

### Provider Overloaded (529)

```json
{
  "error": {
    "message": "The selected provider is currently overloaded. Please try again or select a different model.",
    "code": 529
  }
}
```

### Model Not Found (404)

```json
{
  "error": {
    "message": "Model not found",
    "code": 404
  }
}
```

## Retry Strategy

The API implements exponential backoff retries:

- **Initial delay:** 500ms
- **Max attempts:** 3
- **Max delay:** 60 seconds
- **Exponent:** 1.5

The SDKs handle retries automatically. For manual implementations:

```python
import time
import random

def retry_with_backoff(func, max_attempts=3):
    for attempt in range(max_attempts):
        try:
            return func()
        except Exception as e:
            if attempt == max_attempts - 1:
                raise
            delay = min(500 * (1.5 ** attempt), 60000)
            time.sleep(delay / 1000 + random.uniform(0, 0.1))
```

## Streaming Errors

When streaming, errors may be returned as SSE events:

```
event: error
data: {"error": {"message": "...", "code": 429}}
```
